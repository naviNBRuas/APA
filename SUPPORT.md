## 📋 Information to Include

When seeking help, please provide:

### Essential Information
- **APA Version**: Exact version number (e.g., v2.0.0)
- **Operating System**: Linux/macOS/Windows with version
- **Architecture**: AMD64/ARM64/ARM
- **Go Version**: If building from source
- **Installation Method**: Binary download, source build, Docker

### Issue Details
- **Steps to Reproduce**: Clear, numbered steps
- **Expected Behavior**: What you expected to happen
- **Actual Behavior**: What actually happened
- **Error Messages**: Complete error output with context
- **Configuration**: Relevant config files (redact sensitive data)

### Environment Information
- **Hardware Specifications**: CPU, RAM, storage
- **Network Setup**: Firewalls, proxies, special configurations
- **Related Software**: Other agents, monitoring tools
- **Logs**: Debug-level logs when possible

## 🎯 Support Triage Process

### Response Times
- **General Issues**: Within 2 business days
- **Bug Reports**: Within 1 business day
- **Security Issues**: Within 4 hours (business hours)

### Issue Prioritization
1. **Critical**: Data loss, security vulnerabilities, complete failures
2. **High**: Major functionality broken, performance regressions
3. **Medium**: Feature requests, usability improvements
4. **Low**: Documentation updates, minor enhancements

### Triage Workflow
1. **Initial Review**: Within 24 hours of submission
2. **Classification**: Bug, feature request, question, or support
3. **Assignment**: Appropriate maintainer or community member
4. **Resolution**: Fix, workaround, or explanation provided
5. **Follow-up**: Confirmation of resolution with user

## 🛠️ Troubleshooting Quick Start

### Common Issues Checklist

**Agent Won't Start**:
1. Check configuration file syntax
2. Verify port availability
3. Confirm file permissions
4. Review system logs

**Networking Problems**:
1. Test connectivity to bootstrap peers
2. Check firewall rules
3. Verify DNS resolution
4. Confirm certificate validity

**Performance Issues**:
1. Monitor system resources
2. Check for resource limits
3. Review configuration settings
4. Analyze performance metrics

### Diagnostic Commands

```bash
# Check agent status
./apa --version

# Run with verbose logging
./apa --log-level=debug

# Test configuration
./apa --config=config.yaml --dry-run

# Generate system report
./apa diagnostics
```
