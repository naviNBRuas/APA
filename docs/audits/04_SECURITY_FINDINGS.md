## S-01: Code Regeneration — Remote Code Execution (CRITICAL)

**File**: `pkg/regeneration/strategies.go:50-120`  
**Type**: OWASP [A03:2021 – Injection](https://owasp.org/Top10/A03_2021-Injection/)

**Description**: The `CodeRegeneration` strategy executes `ollama run` via `os/exec` with a hardcoded system prompt that instructs the LLM to rewrite Go source files. The LLM output is written directly to disk without syntax validation, type checking, or security review.

```go
// Pseudocode of the dangerous pattern:
cmd := exec.Command("ollama", "run", modelName)
cmd.Stdin = strings.NewReader(systemPrompt)
output, _ := cmd.Output()
os.WriteFile(targetPath, output, 0644)
```

**Risk**: A prompt injection (e.g., via a crafted dependency file or environment variable) could cause the LLM to produce malicious Go code that would be compiled and executed.

**Recommendation**: Remove `CodeRegeneration` strategy immediately, or gate behind an explicit `--dangerous-enable-code-regeneration` flag and add mandatory output validation (syntax check, compilation test, human review).

## S-02: Linter Security Exclusions (HIGH)

**File**: `.golangci.yml`  
**Type**: OWASP [A05:2021 – Security Misconfiguration](https://owasp.org/Top10/A05_2021-Security-Misconfiguration/)

The linter configuration explicitly excludes these security-relevant checks:

| Linter | Rule | Risk |
|--------|------|------|
| `G404` | Insecure random number generator (`math/rand` vs `crypto/rand`) | Predictable crypto keys/TLS certs |
| `G304` | File path provided as user input | Path traversal |
| `G107` | URL constructed from user input | SSRF |
| `G306` | Poor file permissions (`0644` instead of `0600`) | Credential exposure |
| `G301`/`G302` | Poor directory/file permissions | Unauthorized access |

**Recommendation**: Remove exclusions incrementally. Start with G404 (replace `math/rand` with `crypto/rand` in security contexts), then G306 (fix file permissions), then G304/G107 (add input validation).

## S-03: Admin API Key in Repo (HIGH)

**File**: `configs/agent-config.yaml:15`

```yaml
admin_api_key: "your-admin-api-key-here"
```

**Risk**: While this is a placeholder, having any API key in a configuration file checked into the repository normalizes the pattern. A real key could be committed accidentally.

**Recommendation**: Move `admin_api_key` to environment variable (`APA_ADMIN_API_KEY`). Add `configs/*.yaml` to `.gitignore` and provide `agent-config.yaml.example` instead.

## S-04: Static TLS Serial Number (HIGH)

**File**: `pkg/security/tls.go:45`

```go
serialNumber := big.NewInt(1)
```

**Risk**: Every self-signed certificate generated by APA has the same serial number. This violates RFC 5280 §4.1.2.2 which requires unique serial numbers per certificate. Some TLS implementations may reject certs with duplicate serials.

**Recommendation**:
```go
serialNumberLimit := new(big.Int).Lsh(big.NewInt(1), 128)
serialNumber, _ := rand.Int(rand.Reader, serialNumberLimit)
```

## S-05: Swarm Discovery — No Authentication (HIGH)

**File**: `pkg/swarm/discovery.go`

**Description**: Any peer can announce resources and join the swarm. There is no peer authentication, identity verification, or access control.

**Risk**: A malicious peer could:
- Announce fake resources
- Poison the discovery table
- Launch eclipse attacks
- Exfiltrate resource metadata

**Recommendation**: Add peer identity verification using the existing Ed25519 key infrastructure. Verify peer signatures on all announcements.

---

## S-06: Admin API — Weak Authentication (MEDIUM)

**File**: `pkg/controller/admin/`

**Description**: The admin gRPC API checks for an API key in configuration but has no rate limiting, no audit logging, and no IP-based access control.

**Recommendation**: Add rate limiting, audit logging for all admin operations, and optional mTLS for admin API access.

---

## S-07: No Input Validation on WASM Paths (MEDIUM)

**File**: `pkg/module/`

**Description**: Controller/WASM module paths are read from configuration and used to load files without validation.

**Recommendation**: Add path sanitization, restrict module paths to a configured directory, and verify file signatures before loading.

---

## S-08: Crypto Placeholders (MEDIUM)

**File**: `pkg/security/crypto.go`

**Description**: `Encrypt()` and `Decrypt()` are placeholders with `return nil` bodies. The code imports crypto libraries but has no actual encryption implementation.

**Recommendation**: Implement or remove. If data-at-rest encryption is required, implement AES-256-GCM with key derivation from the agent's Ed25519 key.

---

## S-09: Multiple `panic()` Calls (LOW)

**File**: Throughout codebase

**Description**: Several functions use `panic()` instead of returning errors.

**Recommendation**: Replace all `panic()` calls in production paths with error returns.

---

## S-10: No Secret Rotation (LOW)

**Description**: Agent identity keys are generated once and used for the lifetime of the agent. There is no key rotation mechanism.

**Recommendation**: Add configurable key rotation with grace periods for connection draining.
